# Audit i18n complet — 184 outils × 20 langues (10 juin 2026)

## Périmètre
Audit demandé : « vérifier dans les moindres détails que les 184 modules fonctionnent
parfaitement dans toutes les langues, sans incohérence de langue ».

Méthode :
1. **Audit statique** — croisement catalogue (`TOOLS`) ↔ routes EN ↔ routeur localisé
   ↔ registres (`FFMPEG_TOOLS`, `TEXT_TOOLS`, `IMAGE_TOOLS`, `AI_TEXT_TOOLS`) ↔ `FN_MAP`
   ↔ couches i18n (`TOOL_TRANSLATIONS` + `GENERATED_TOOL_TRANSLATIONS`).
2. **Audit dynamique** — build de production + crawl HTTP réel de **3 900 pages**
   (184 outils × 20 langues + 10 catégories × 20 langues + 20 accueils) avec
   vérification : statut 200, `<html lang>`, `dir="rtl"` (arabe), H1 localisé attendu.

## Résultats AVANT corrections
| Constat | Ampleur | Gravité |
|---|---|---|
| `<html lang="en">` sur TOUTES les pages traduites (rustine JS post-chargement) | 3 496 pages | 🔴 |
| `dir="rtl"` absent du rendu serveur en arabe | 184 pages | 🔴 |
| 10 outils sans AUCUNE traduction (5 sécurité + 5 liens) → 100 % anglais hors EN | 190 page-langues | 🟠 |
| Pages cassées / 404 / H1 manquant / mauvais contenu | **0** | ✅ |
| Routeur localisé : composant erroné | **0** | ✅ |

## Corrections appliquées
1. **`middleware.ts`** : la locale de l'URL est transmise au layout racine via l'en-tête
   `x-locale` (même mécanisme que le nonce CSP).
2. **`app/layout.tsx`** : `<html lang={locale} dir={rtl}>` rendu CÔTÉ SERVEUR.
   Le layout lisait déjà `headers()` (nonce) donc aucun coût de génération supplémentaire.
   La rustine client `HtmlLang` est conservée pour les routes non préfixées pilotées
   par cookie.
3. **`lib/i18n/tool-translations.ts`** : traductions manuelles complètes
   (name / short / h1 / metaTitle / metaDescription × 19 langues) pour :
   `ssl-checker`, `password-checker`, `ssh-key-generator`, `text-encryptor`,
   `file-encryptor`, `url-shortener`, `deep-link`, `magic-link`, `utm-builder`,
   `link-builder` — soit ~950 chaînes.

## Résultats APRÈS corrections (re-crawl complet)
- **3 680 / 3 680** pages outils : ✅ 0 défaut (statut, lang, dir, H1 localisé).
- **220 / 220** pages catégories + accueils : ✅ 0 défaut.
- **0** couple outil×langue sans traduction (était 190).
- `tsc --noEmit` : 0 erreur. `next build` : succès.

## Limites connues (hors périmètre de ce correctif)
- **Libellés internes des widgets** (boutons « Download », placeholders…) : une partie
  des composants clients reste en anglais quelle que soit la langue de la page — motif
  historique de la plateforme (le contenu SEO/page est localisé, le widget partiellement).
  Chantier séparé si souhaité : généraliser `lib/i18n/tool-ui.ts`.
- **Dérive de langue du résumeur IA** (résumé d'un texte EN renvoyé en FR) : bug backend
  connu de `ai-process` (3 tentatives de fix prompt v29/v30/v32) ; non testable depuis ce
  sandbox (réseau sortant bloqué vers Supabase). À retester en conditions réelles.
- Le script `scripts/fill-translations.mjs` n'a pas pu tourner (réseau) ; les 10 outils
  ont été traduits À LA MAIN dans la couche prioritaire `TOOL_TRANSLATIONS` (qualité ≥ machine).
